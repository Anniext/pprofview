# 避免生成不必要的文档

## 核心原则

在处理代码实现、bug 修复、功能开发等任务时，**不要自动生成、创建或更新任何文档**，除非用户明确要求。

**重要**：
- 专注于解决问题和实现需求
- 不要主动创建或更新 README.md、CHANGELOG.md 等任何文档
- 只有当用户明确要求时才创建或更新文档
- 完成任务后用简短的口头总结即可

## 适用场景

### ❌ 不要生成文档的情况

1. **代码实现完成后**
   - 不要生成 `IMPLEMENTATION_SUMMARY.md`
   - 不要生成 `CHANGES.md`
   - 不要生成 `WORK_SUMMARY.md`

2. **Bug 修复后**
   - 不要生成 `BUG_FIX_REPORT.md`
   - 不要生成 `FIX_SUMMARY.md`

3. **功能添加后**
   - 不要生成 `FEATURE_SUMMARY.md`
   - 不要生成 `NEW_FEATURES.md`

4. **重构完成后**
   - 不要生成 `REFACTORING_REPORT.md`

### ✅ 应该生成文档的情况

只在以下情况下生成文档：

1. **用户明确要求**
   - "请生成一份文档说明..."
   - "帮我写一个 README..."
   - "创建使用指南..."

2. **项目必需的文档**
   - README.md（项目初始化时）
   - API 文档（开发 API 时）
   - 用户手册（开发用户功能时）

3. **技术设计文档**
   - 仅在设计阶段或用户要求时

## 正确的工作流程

### 代码实现任务

```
1. 理解需求
2. 实现代码
3. 编写必要的代码注释
4. 运行测试验证
5. 简短口头总结（1-2 句话）
   ✅ "已完成 Goc 集成功能，添加了运行配置扩展和 UI 编辑器。"
   ❌ 不要创建 IMPLEMENTATION_SUMMARY.md
```

### Bug 修复任务

```
1. 定位问题
2. 修复代码
3. 验证修复
4. 简短口头说明
   ✅ "已修复空指针异常，添加了空值检查。"
   ❌ 不要创建 BUG_FIX_REPORT.md
```

### 功能开发任务

```
1. 设计功能
2. 实现代码
3. 测试功能
4. 简短口头总结
   ✅ "已添加 Goc 集成功能。"
   ❌ 不要创建 FEATURE_IMPLEMENTATION.md
   ❌ 不要更新 README.md 或 CHANGELOG.md（除非用户明确要求）
```

## 文档更新原则

### 只在用户明确要求时才更新文档

**不要主动更新**：
- CHANGELOG.md
- README.md
- 任何其他项目文档

**应该编写的**：
- 代码注释（函数和类的文档注释）
- 复杂逻辑的行内注释

### 不要创建的文档

1. **工作总结类**
   - IMPLEMENTATION_SUMMARY.md
   - WORK_REPORT.md
   - TASK_COMPLETION.md

2. **过程记录类**
   - DEVELOPMENT_LOG.md
   - CODING_PROCESS.md
   - STEP_BY_STEP.md

3. **重复性文档**
   - 与 README 重复的内容
   - 与代码注释重复的内容
   - 与 CHANGELOG 重复的内容

## 口头总结原则

完成任务后，使用简短的口头总结：

### ✅ 好的总结

```
已完成 Goc 集成功能：
- 添加了运行配置扩展
- 实现了 UI 编辑器
```

### ❌ 不好的做法

```
创建了以下文档：
- docs/IMPLEMENTATION_SUMMARY.md
- docs/TECHNICAL_DESIGN.md
- docs/USAGE_GUIDE.md
- docs/QUICK_START.md
- docs/FAQ.md
```

## 例外情况

以下情况可以生成文档，但需要用户明确同意：

1. **大型功能开发**
   - 询问："是否需要创建详细的技术文档？"

2. **公共 API 开发**
   - 询问："是否需要生成 API 文档？"

3. **用户功能**
   - 询问："是否需要创建用户使用指南？"

## 总结

**核心思想**：代码即文档，注释即说明。不要为了"完整性"而创建冗余的 Markdown 文档。

**记住**：
- 代码完成 = 任务完成
- 简短总结 > 任何文档
- 不要主动创建或更新任何文档
- 只有用户明确要求时才处理文档

## 检查清单

在完成任务前问自己：

- [ ] 用户是否明确要求生成文档？
- [ ] 这个文档是否是项目必需的？
- [ ] 这个文档是否会被实际使用？
- [ ] 这个文档的内容是否已经在其他地方存在？

如果以上问题的答案都是"否"，那就不要生成文档。
